Crea un formulario HTML de registro para un nuevo usuario tipo empleador.

Debe incluir los siguientes campos:

Datos personales:

Nombre y apellido (obligatorio, solo letras y espacios)

Email (obligatorio, único, validación formato email)

Teléfono (obligatorio, solo números)

Datos del negocio:

Nombre del kiosco (obligatorio)

País (select desplegable con lista de países)

Provincia/Estado (select dinámico según país)

Distrito

Localidad

Domicilio del negocio

Plan:

Selección de plan (radio buttons o select):

Prueba

Standard

Premium

Validaciones:

Todos los campos obligatorios

Email válido

Teléfono solo numérico

No permitir envío si hay errores

Mostrar mensajes debajo de cada input

Al enviar el formulario:

Enviar los datos mediante fetch POST al backend

El backend guardará los datos en:

colección usuarios/{uid}

colección negocios/{nid}

Modelo de datos:

usuarios/{uid}
{
plan: string,
vencimientoPlan: timestamp,
activo: true,
createdAt: timestamp,
displayName: string,
email: string,
role: "empleador",
tenantId: string,
uid: string,
updatedAt: timestamp
}

negocios/{nid}
{
nid: string,
createdAt: timestamp,
displayName: string,
email: string,
pais: string,
provinciaEstado: string,
contacto: number,
distrito: string,
localidad: string,
domicilio: string,
tenantId: string,
uid: string,
updatedAt: timestamp
}

Generar:

HTML + CSS moderno

JS para validaciones

Función fetch lista para conectar con backend Node.js + Firebase


arquitectura actual:
Frontend (Vercel):

Formulario HTML

Validaciones

fetch POST → Firebase Function

Firebase Function:

Verifica datos

Crea usuario en Firebase Auth

Genera tenantId

Calcula vencimiento

Guarda en:

usuarios/{uid}

negocios/{nid}

Firestore:

Solo acepta escrituras desde Functions

Reglas bloqueando acceso directo

⚠️ Lo que NO deberías hacer

❌ Guardar plan desde frontend sin validación
❌ Permitir que el frontend defina el vencimiento
❌ Crear tenantId en el cliente
❌ Confiar en datos sin verificar
